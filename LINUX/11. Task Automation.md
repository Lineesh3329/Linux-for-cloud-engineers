## Task Automation
## Cron (Job Scheduling)
Cron is commonly used for backups, log cleanup, running scripts, and maintenance tasks in servers.
Cron is a Linux utility used to schedule tasks automatically at specific times or intervals.

#### Time Shedule

    Every Min                    
    Every 5 mins
    Every Hour
    Every Day
    Every Sunday
    Every Month

#### Cron Time Format
    
    * * * * * command
    | | | | |
    | | | | └── Day of Week  (0-7)
    | | | └──── Month        (1-12)
    | | └────── Day of Month (1-31)
    | └──────── Hour         (0-23)
    └────────── Minute       (0-59)

#### Main Symbols 
    '*'  - Every
    '/'  - Step
    ','  - Multiple values
    '-'  - Range
    
#### Cron Time cheat sheet

    *              Run Every
    */2            Run Every 2 mins
    0              Run Exact Minute
    0 0 * * *      Run Everyday at Midnight
    */10 * * *     Run Every 10 mins
    0*/6 * * *     Run Every 6 hrs
    0 0 * * *      Run Everyday at Midnight
    0 0 * * 0      Run Every Sunday
    0 0 1 * *      Run Every month on day 1

##### Where are cron jobs stored ? In the crontab file.

    crontab -e              - Add/edit cron jobs  (open)
    crontab -l              - List cron jobs      (view)
    crontab -r              - Remove cron jobs
    systemctl status cron   - Check cron service

Cron is used to automate tasks in Linux. It is very important in server administration and cloud environments.
